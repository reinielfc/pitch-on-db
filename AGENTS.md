## Project Overview

- **Backend**: Go REST API with Gin framework and PostgreSQL
- **Frontend**: SvelteKit 5 with Tailwind CSS and DaisyUI

## Backend Architecture

### Layered Structure
```
HTTP Request
    â†“
routes.Register() â†’ handlers/ (HTTP handlers)
    â†“
services/ (Business logic - INTERFACES)
    â†“
repos/ (Data access - INTERFACES)
    â†“
db/ (Generated sqlc queries)
    â†“
PostgreSQL
```

### Core Principles
1. **Interface-driven design**: All services and repos are interfaces for testability
2. **Dependency injection**: Constructor functions pass dependencies (e.g., `NewPigeonService(pigeonRepo, tagsRepo)`)
3. **Domain-driven errors**: Use `domain.DomainError` with error codes
4. **Middleware pattern**: HTTP middleware for logging, error handling, validation

### Key Packages

| Package | Purpose |
|---------|---------|
| `domain/` | Core types, error codes, interfaces |
| `services/` | Business logic (INTERFACE contracts) |
| `repos/` | Data access (INTERFACE contracts) |
| `handlers/` | HTTP request handlers |
| `routes/` | Route registration with handlers |
| `db/` | Generated by sqlc (never edit manually) |
| `config/` | Environment-based configuration |
| `middleware/` | HTTP middleware (errors, logging, validation) |

## Code Patterns & Conventions

### 1. Service Interface Pattern

**File**: `apps/api/services/pigeon.go`

```go
// Define interface at package level
type PigeonService interface {
    Get(ctx context.Context, id int64) (*domain.Pigeon, error)
    Create(ctx context.Context, pigeon *domain.Pigeon) (*domain.Pigeon, error)
    // ...
}

// Private struct with dependencies via injection
type pigeonService struct {
    pigeonRepo repos.PigeonRepository
    tagsRepo   repos.TagRepository
    // ... logger, etc.
}

// Constructor
func NewPigeonService(pigeonRepo repos.PigeonRepository, tagsRepo repos.TagRepository) PigeonService {
    return &pigeonService{
        pigeonRepo: pigeonRepo,
        tagsRepo:   tagsRepo,
    }
}

// Receiver methods on private struct
func (svc *pigeonService) Get(ctx context.Context, id int64) (*domain.Pigeon, error) {
    // Implementation
}
```

**Key rules**:
- Service interfaces are **public**, struct implementations are **private** (lowercase)
- All methods take `context.Context` as first parameter
- Constructor: `NewServiceName()`
- Receiver methods on private struct

### 2. Repository Interface Pattern

**File**: `apps/api/repos/pigeon.go`

```go
type PigeonRepository interface {
    Get(ctx context.Context, id int64) (*domain.Pigeon, error)
    Create(ctx context.Context, pigeon *domain.Pigeon) (*domain.Pigeon, error)
    // ...
}

type pigeonRepository struct {
    queries *db.Queries
}

func NewPigeonRepository(queries *db.Queries) PigeonRepository {
    return &pigeonRepository{queries: queries}
}

func (r *pigeonRepository) Get(ctx context.Context, id int64) (*domain.Pigeon, error) {
    // Use r.queries (generated by sqlc)
}
```

**Key rules**:
- Repos interface with underlying data source (sqlc queries, external APIs, etc.)
- All methods take `context.Context` as first parameter
- Use dependency injection for queries

### 3. Error Handling with domain.DomainError

**File**: `apps/api/domain/errors.go`

```go
// Use these patterns:

// Simple operation error - wraps with context
return domain.Errorf("Create: %w", err)  // Becomes domain.ErrInternal

// Not found error (404)
return domain.NewResourceNotFoundError("pigeon", id)

// Validation error (400)
return domain.NewInvalidError("band_number must not be empty")

// Conflict error (409)
return domain.NewConflictError("pigeon with this band_number already exists", domain.WithCtx("band_number", bandNum))

// Custom error codes are in domain/errors.go
```

**Error codes**:
- `ErrNotFound` â†’ 404
- `ErrInvalid` â†’ 400
- `ErrConflict` â†’ 409
- `ErrInternal` â†’ 500

### 5. Route Registration

**File**: `apps/api/routes/routes.go`

```go
func Register(engine *gin.Engine, pigeonSvc services.PigeonService, tagsSvc services.TagsService) {
    pigeonHandler := handlers.NewPigeonHandler(pigeonSvc)
    tagsHandler := handlers.NewTagsHandler(tagsSvc)

    pigeons := engine.Group("/api/pigeons")
    {
        pigeons.GET(":id", pigeonHandler.Get)
        pigeons.POST("", pigeonHandler.Create)
    }
    
    tags := engine.Group("/api/tags")
    {
        tags.GET("", tagsHandler.List)
        tags.POST("", tagsHandler.Create)
    }
}
```

## Database Layer

### sqlc Code Generation

- **Config**: `database/sqlc.yml` configures output to `apps/api/db/`
- **Queries**: Define SQL in `database/queries/pigeons.sql` and `tags.sql`
- **Run**: `make generate` produces `apps/api/db/*.go` (type-safe query functions and models)
- **NEVER edit** generated `db/*.go` files directly

### Migrations with Goose

**Location**: `database/migrations/` (e.g., `0001_pigeons.sql`)

```sql
-- +goose Up
CREATE TABLE pigeons (
    id SERIAL PRIMARY KEY,
    name VARCHAR NOT NULL,
    band_number VARCHAR UNIQUE,
    birth_date DATE,
    sex CHAR(1),
    created_at TIMESTAMP DEFAULT NOW()
);

-- +goose Down
DROP TABLE pigeons;
```

**Run migrations**: `make migrate`

## Testing Patterns

### Test File Location & Naming
- **File**: `apps/api/services/pigeon_test.go` (same directory, `package services_test`)
- **Naming**: Test functions `TestServiceName_MethodName`
- **Pattern**: Table-driven tests with `t.Run()` subtests

### Table-Driven Test Example

**File**: `apps/api/services/pigeon_test.go`

```go
package services_test

import (
    "context"
    "testing"
    
    "github.com/stretchr/testify/assert"
    "github.com/stretchr/testify/mock"
    
    "pitch-on-db/domain"
    "pitch-on-db/services"
    "pitch-on-db/services/mocks"
)

func TestPigeonService_Get(t *testing.T) {
    tests := []struct {
        name    string
        pigeonID int64
        mockSetup func(*mocks.PigeonRepository)
        expectedPigeon *domain.Pigeon
        expectedErr error
    }{
        {
            name:     "success: get existing pigeon",
            pigeonID: 42,
            mockSetup: func(repo *mocks.PigeonRepository) {
                repo.On("Get", mock.Anything, int64(42)).
                    Return(&domain.Pigeon{ID: 42, Name: "Flutters"}, nil).
                    Once()
            },
            expectedPigeon: &domain.Pigeon{ID: 42, Name: "Flutters"},
            expectedErr: nil,
        },
        {
            name:     "error: not found",
            pigeonID: 999,
            mockSetup: func(repo *mocks.PigeonRepository) {
                repo.On("Get", mock.Anything, int64(999)).
                    Return(nil, domain.NewResourceNotFoundError("pigeon", 999)).
                    Once()
            },
            expectedPigeon: nil,
            expectedErr: domain.ErrNotFound,
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            deps := setupPigeonServiceDeps(t)
            tt.mockSetup(deps.Repo)
            
            svc := services.NewPigeonService(deps.Repo, deps.TagsRepo)
            pigeon, err := svc.Get(context.Background(), tt.pigeonID)
            
            deps.Repo.AssertExpectations(t)
            if tt.expectedErr != nil {
                assert.ErrorIs(t, err, tt.expectedErr)
            } else {
                assert.NoError(t, err)
                assert.Equal(t, tt.expectedPigeon.ID, pigeon.ID)
            }
        })
    }
}

func setupPigeonServiceDeps(t *testing.T) struct {
    Repo *mocks.PigeonRepository
    TagsRepo *mocks.TagRepository
} {
    return struct {
        Repo *mocks.PigeonRepository
        TagsRepo *mocks.TagRepository
    }{
        Repo: mocks.NewPigeonRepository(t),
        TagsRepo: mocks.NewTagRepository(t),
    }
}
```

### Generate Mocks
```bash
make generate  # Creates mocks in services/mocks/ from interfaces
```

**Key testing rules**:
- Use `testify/assert` for assertions
- Use `testify/mock` and generated mocks for dependencies
- Mock setup: `.On("MethodName", args).Return(result, err).Once()`
- Assert: `deps.Repo.AssertExpectations(t)`

## When Adding New Features

### 1. **Add Database Schema**
   - Create migration in `database/migrations/` (e.g., `0005_new_table.sql`)
   - Run `make migrate`

### 2. **Add SQL Queries**
   - Write queries in `database/queries/pigeons.sql` or `tags.sql`
   - Run `make generate` to create sqlc code in `apps/api/db/`

### 3. **Implement Repository**
   - Create `repos/new_thing.go` with `ThingRepository` interface
   - Implement with `thingRepository` struct using `db.Queries`
   - Follow the pattern in `repos/pigeon.go`

### 4. **Implement Service**
   - Create `services/thing.go` with `ThingService` interface
   - Implement with `thingService` struct, inject dependencies
   - Follow the pattern in `services/pigeon.go`

### 5. **Add Tests**
   - Create `services/thing_test.go` with table-driven tests
   - Generate mocks: `make generate`
   - Follow patterns in `services/pigeon_test.go`

### 6. **Create HTTP Handler**
   - Create `handlers/thing.go` with `ThingHandler` struct
   - Inject `ThingService` in constructor
   - Follow patterns in `handlers/pigeons.go`

### 7. **Register Routes**
   - Update `routes/routes.go` to add handler and register endpoints
   - Follow Gin grouping convention

### 8. **Frontend Integration** (SvelteKit)
   - Add route in `apps/web/src/routes/`
   - Use `fetch('/api/endpoint')` to call backend
   - Leverage DaisyUI components for UI

## Common Pitfalls & Tips

1. **sqlc changes**: Never edit `db/*.go` files directly â€” regenerate with `make generate`
2. **Mock registration**: All mock setup calls require `.Once()` or `.Times(n)` for assertion
3. **Context passing**: All service/repo methods require `context.Context` as first parameter
4. **Error types**: Always wrap errors with `domain.Errorf()` or use specific error constructors
5. **Hot reload**: After generating code (sqlc, mocks), the API container auto-reloads
6. **Port conflicts**: Ensure ports 5173 (web), 8080 (api), 5432 (db) are available locally
